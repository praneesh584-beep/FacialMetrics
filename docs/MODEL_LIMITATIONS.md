# Model Limitations

FaceMetric does not contain a validated attractiveness model.

The development mock score is deterministic, demo-only, disabled from production intent, and not scientifically validated. It must not be presented as a real attractiveness rating.

A real preference model would require:

- Lawful and ethical data collection.
- Meaningful informed consent.
- Clear rating instructions.
- Appropriate rater representation.
- Bias analysis.
- Train, validation, and held-out test splits.
- Reliability measurements.
- Calibration.
- Model cards.
- Versioning.
- User-facing uncertainty.
- Independent validation.
- Privacy and legal review.

FaceMetric must not infer protected or sensitive traits from biometric data.
